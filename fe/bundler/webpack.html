webpack 学习总结 ​

webpack初体验 ​

shell
pnpm i webpack webpack-cli -D

webpack默认会寻找以src命名的文件，找不到则找src目录下的index命名的作为入口文件

webpack.config.js ​

js
const path = require('path')

module.exports = {
  entry: './src/main.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js',
  },
}

webpack工作模式 ​

  • 默认webpack有3种工作模式，分别为production、development、none
  • 可以通过命令行参数以及webpack配置文件显示指定模式，默认值为production

webpack loader ​

  • webpack默认情况下会把它看到的所有文件都视为js文件加载
  • 故而，如果我们需要加载其他后缀名的文件，我们需要配置——loader

css文件 ​

JS
//webpack.config.js

const path = require('path')

module.exports = {
  module: {
    rules: [{ test: /\.css$/, use: ['style-loader','css-loader'] }],
  },
}

webpack中的css-loader会将css文件打包成js代码，style-loader会以js代码形式将css插入页面的header的style标签里

其他类型文件 ​

对于那些不好被打包成js代码的文件，我们可以使用file-loader对其进行处理，此loader原理则是将匹配到的文件copy一份到输出文件夹里

js
//webpack.config.js

const path = require('path')

module.exports = {
  module: {
    rules: [
      { test: /\.css$/, use: ['style-loader','css-loader'] },
      { test: /\.png$/, use: ['file-loader'] },
    ],
  },
}

publicPath ​

url-loader ​

Data Url是一种特殊的协议，这种url里直接就包含着内容，浏览器可以根据此协议解析出文件内容

webpack里我们可以通过url-loader将匹配到的文件转换为data url插入到js代码里

js
module.exports = {
  module: {
    rules: [
      // { test: /\.png$/, use: ['file-loader'] },
      {
        test: /\.png$/,
        use: {
          loader: 'url-loader',
          options: {
            //配置limit大于10kb的交给file-loader处理，小于10kb还是url-loader处理
            limit: 10 * 1024 // 10kb
          }
        }
      },
    ],
  },
}

loader类别 ​

  • 编译转换类 css-loader
  • 文件操作类 file-loader
  • 代码检查类 eslint-loader

bable-loader ​

webpack对js文件默认的加载器并不能实现语法降级效果，babel-loader可以做到

js
//webpack.config.js

const path = require('path')

module.exports = {
  module: {
    rules: [
      { 
        test: /\.js$/, 
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        } 
      },
    ],
  },
}

html-loader ​

自己实现一个markdown-loader ​

js
const marked = require('marked')

module.exoprts = source => {
  const html = marked(source)
  return `module.exports = ${JSON.stringify(html)}`
}

webpack核心工作原理 ​

以入口文件为起点，根据文件的导入导出关系，形成一个模块的依赖树，最后遍历或者说递归这个依赖树，打包成最终的文件

webpack插件 ​

js

  plugins: [
    new CleanWebpackPlugin()

html-webpack-plugin ​

输出html文件

js

const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      title: 'Webpack Study',
      template: './src/index.html'
    })
  ]
}

copy-webpack-plugin ​

将配置的目录拷贝到输出目录

此plugin与file-loader的区别： loader是加载资源文件的，file-loader虽然和copy-webpack-plugin都可以将资源文件拷贝到输出目录，但是file-loader 会对资源文件进行处理，而此plugin不会。故而，一个是loader，一个是插件。

js
//webpack.config.js
const CopyWebpackPlugin = require('copy-webpack-plugin')

module.exports = {
  plugins: [
    new CopyWebpackPlugin({
      patterns: [
        { from: 'public', to: 'public' },
      ]
    })
  ]
}

source map ​

js
//webpack.config.js

module.exports = {
  devtool: 'source-map',
}

配置devtool的不同取值：

  • source-map 生成source-map文件，正儿八经的使用source-map方法
  • eval 不会生成source-map文件 可以定位到错误的文件名
  • eval-source-map 生成source-map文件 可以定位到错误的文件以及行列信息
  • eval-cheap-source-map 生成source-map文件 可以定位到错误的文件以及行信息，没有列信息
  • eval-cheap-module-source-map 生成source-map文件 可以定位到错误的文件以及行信息，没有列信息，错误位置代码为经过loder加工
  • ......

怎么选择source map？

  • 开发建议选择eval-cheap-module-source-map
  • 生产环境选择none，不建议使用source map
  • 如果生产一定要使用，建议选择nosources-source-map

webpack自动刷新的问题 ​

不能保留我们在浏览器那里进行的状态 HMR可以帮我们解决这个问题

HMR ​

默认即支持js、css等文件的热更新，即HMR

可配置hot："only"来阻止浏览器的自动刷新

生产环境优化 ​

  • 配置文件根据环境不同导出不同配置
js
module.exports = (env, argv) => {
  if(env === 'production') {
    return {
      mode: 'production'
    }
  }
  return {
  }
}

  • 一个环境对应一个配置文件

  • webpack.common.js

  • webpack.dev.js

  • webpack.prod.js

你可以通过 --merge 选项来合并两个或多个不同的 webpack 配置（相同配置以后面配置为准）：

npx webpack --config ./first.js --config ./second.js --merge

同时也可使用webpack-merge模块提供的方法来合并配置

webpack definePlugin ​

可通过此插件来为项目中注入全局变量

js
//webpack.config.js
const path = require('path')
const webpack = require('webpack')

module.exports = {
  plugins: [
    new webpack.DefinePlugin({
      VERSION: JSON.stringify('1.0.0'),
    })
  ],
}

Tree Shaking ​

Tree Shaking不是一个配置，而是webpack一些功能的集合

js
//webpack.config.js
module.exports = {
  optimization: {
    usedExports: true,  //仅导出使用到的变量或者函数等
    minimize: true,   //压缩
    concatenateModules: true, //尽可能的将所有模块合并输出到一个函数中
  },
}

Tree Shaking是基于esmodule的，如果使用babel-loader将代码中的esm转换成了cjs再交给webpack处理，那么Tree Shaking则可能失效

webpack SideEffects ​

js
//webpack.config.js
module.exports = {
  optimization: {
    sideEffects: true
  },
}

开启了sideEffects之后webpack会检查项目中的package.json文件中的配置，从而判断哪些文件没有副作用，从而根据配置文件来tree shaking

多入口打包 ​

js
const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
  mode: 'development',
  entry: {
    // 配置两个入口
    index: './src/index.js',
    album: './src/album.js'
  },
  output: {
    filename: '[name].bundle.js'
  },
  plugins: [
    new HtmlWebpackPlugin({
      title: 'Multi Entry',
      template: './src/index.html',
      filename: 'index.html',
      chunks: ['index']
    }),
    new HtmlWebpackPlugin({
      title: 'Multi Entry',
      template: './src/album.html',
      filename: 'album.html',
      chunks: ['album']
    }),
  ]
}

};

hash ​

js
const MiniCssExtractPlugin = require('mini-css-extract-plugin')

module.exports = {
  mode: 'production',
  output: {
    filename: '[name]-[hash].bundle.css'
  },
  module: {
    rules: [
      { test: /\.css$/, use: [ MiniCssExtractPlugin.loader, 'css-loader'] },
    ],
  },
  plugins: [
    new MiniCssExtractPlugin({
      filename: '[name]-[hash].bundle.css'
    })
  ]
}

filename的3个值：

  • [name]-[hash].bundle.css 项目类型的hash ，项目中有任一变化，hash就变
  • [name]-[chunkhash].bundle.css 打包过程中，同一路的打包，chunkhash都是相同的，如果有代码变化，则对应chunkhash变化
  • [name]-[contenthash].bundle.css 文件级别的hash， 不同文件生产的hash不同

同时，若是嫌hash长，也可通过占位符的方式指定hash的长度 [name]-[contenthash:8].bundle.css
